K Nearest Neighbor

It is an algorithm that I coded as part of a group project withe simon where we had to develop software to sort news dispatches so journalists could present them more easily. The algorithm works relatively: instead of sorting by specific keywords like "sports" or "science", we used a reference dispatch for each category and measured how close a new dispatch was to each one. I really enjoyed researching how this algorithm works and how to implement it. Even though it was a small, optional part of the project, it was important to meet the client's needs in terms of both accuracy and speed.

Assembly game

This game was fun to make. It was developed in assembly language for the Motorola 6800 processor, which was used in some Mega Drive consoles. It was a bit challenging because, unlike higher-level programming, you have to think about the smallest structures, like loops, and I had to manually manage data in RAM due to the limited memory. It was also my first introduction to optimization — for example, we avoided using multiplication for technical reasons. Still, it was really fun, and I could see myself doing more of this kind of work, especially if it involves graphics cards. I have to mention that it was very demanding for me, and I had to put in twice the hours others did — but I don’t regret it.
